Slot Machine

Creates a fully customizable slot machine that can be played by your twitch chat and displayed in OBS

Features

  • Slot Images are fully customizable
  • Rewards/Prizes are also fully customizable
  • Ability to spin all 3 slots or specific slots
  • Ability to add rewards to the prize pool dynamically
  • Ability to change the slot image configuration on the fly
  • Comes fully setup with a bank of images and default settings

Setup

  1. Copy the entire Slots folder into your SAMMI install location
  2. Install the extension. Refer to the SAMMI documentationon how to install an extension.
  3. Ensure OBS is open and connected to SAMMI
  4. The extension will now run the setup and create the required scenes and sources
  5. When the setup has complete, you should see a popup like the below:

Settings

You can enter the Settings button in the deck and customize any settings.
If you change any settings, please restart SAMMI afterwards for the new settings to be applied.
Setting Description
broadcaster_name Your Twitch name, only change this if you are using a bot account
symbols_only_from_left Enable left-to-right-only wins
min_rest_spins Minimum spins before slot fully stops
max_rest_spins Maximum spins before slot fully stops

Customizing rewards

You can customize any rewards - the rewards listed here are all setup for the default images and symbols
Value Description
Prize Name The name of the prize the user will win
Symbol Name The name of the symbol (By default: Cherry, Diamond, Orange, Grapes, Melon)
Amount of Symbols Amount of matching Symbols needed to win this prize
Extension Trigger This is optional and can be left empty
This will automatically trigger an extension trigger with this name when someone gets this
  • There are predefined Prize buttons with all of the Prize triggers already set, on the right hand side of the deck.
    You can add anything in the respective buttons if you want something to happen for each win
  • By default all of the slot chat message commands are turned off in the respective Prize buttons in the deck.
    If you want to turn on any messages for different prizes, you can do so in each button
  • After you change any settings or rewards, restart SAMMI

Using the Extension

  • After you have setup the extension to your liking, you can use it "out of the box" by adding any triggers to the Spin the Slot Machine button.
    By default !spin is enabled for the broadcaster only.
  • If you want a more custom setup you can edit the images and symbols used by adding the images to the images folder within the Slots folder that you copied into your SAMMI install location

    The images MUST be in the format <number>_<name>.png
    If any numbers are missing they will be skipped

    You are free to add multiple of the same images. The more of them there are, the higher the chance of landing on that image increases.

    The rotation of the slot wheel is always in the same order that you number the images in the folder.
    For example, the image would start at some random position, let's say Diamond and then swap to the next image.

    If your list looked like 1_Cherry, 2_Diamond, 3_Orange, 4_Cherry, 5_Grapes, 6_Cherry, 7_Melon, 8_Orange
    The rotation would go Diamond > Orange > Cherry > Grapes > Cherry > Melon > Orange > Cherry > Diamond
    After you've added any images you want, you can edit the symbol names (this is the <name> you entered above) and Prize Names associated with them in the Settings button then restart SAMMI
    • You can then create your own buttons to manage the prizes, if you set an extension trigger for them in the Settings button
    You can use the existing Prize buttons on the right side of the deck as an example.
  • You can add an extension trigger command with the name SLOTS MESSAGE * which will automatically wait 7 seconds and then send your specified chat message (so as not to spoil the result from any delay)
  • If you want to test any rewards you have setup, there is a Slots Test Button that lets you set any combination of slot symbols for testing that the right prize is rewarded and that your extension triggers work

Example Payload

You can use a Trigger Pull Data command with the value All to get the payload object from the trigger

{ "symbol_1": "Grapes",
"symbol_2": "Diamond",
"symbol_3": "Cherry",
"user_name": "TestUser",
"same_amount": 1,
"matching_symbol": "None",
"prize": "1 Grape" }
Field Description
symbol_1 Symbol name for Slot 1
symbol_2 Symbol name for Slot 2
symbol_3 Symbol name for Slot 3
user_name Username of the user who won the prize
same_amount Amount of matching symbols
matching_symbol Symbol name of the matching symbol
prize Prize name entered in the Add Reward command

Extension Commands

Slots: Play

Field Description
User Name The user name to pass to the slot machine
Wait Until Variable Exists You must wait for "SLOTS.is_finished" to be true

Slots: Spin

Field Description
Select Slot Dropdown menu to select: All, Slot 1, Slot 2, Slot 3

Slots: Stop

Field Description
Select Slot Dropdown menu to select: All, Slot 1, Slot 2, Slot 3
Force Stop Yes = Immediately stops the slot; No = Gradually stops after 15–20 symbol changes

Slots: Set Slot

Field Description
Select Slot Dropdown menu to select: All, Slot 1, Slot 2, Slot 3
Value The name of the Symbol

Slots: Show/Hide

Field Description
Option Show or Hide the Slots
User Name Optional – a username to display; if left empty, none is sent

Slots: Add Reward

Field Description
Prize Name The name of the prize the user will win
Symbol Name The name of the symbol
Amount of Symbols Amount of matching Symbols needed to win this prize
Extension Trigger Optional – will trigger an extension trigger with this name if set

Slots: Change Image Array

Field Description
Image Names A list of image names split by a semicolon (;)
Example 2_Diamond.png;7_Melon.png;8_Orange.png
