Discord Webhook

This extension lets you connect SAMMI to a websocket server

This extension lets you connect to a websocket server and receive any responses in SAMMI! Check out the features below!

Features

  • Connect to a websocket server
  • Send a message to the websocket server
  • Listen for any responses from the websocket server via extension trigger
  • Multiple connections are allowed and listed in the bridge for easy viewing
  • Auto reconnect if the connection is dropped
  • Full example deck

Setup

  1. Install the extension. Refer to the SAMMI documentation on how to install an extension.
  2. Check out the example deck for a fully functional test to let you see how things operate.
  3. You can test if the extension has been installed correctly by using the test buttons in the deck. First run the Connect to a testserver button and then confirm that you get a yellow popup in the bottom left of the SAMMI window with a message starting "Data Received:"
    Optionally you can then run the Send a message to the server button and another message should popup in the bottom left "Data Received: Hi"

Using the Commands

  • WS: Start Connection - This will add a new connection to a websocket server.
    • Name - This is the name of the connection. The name used here will become the name of the extension trigger for this connection.
    • Websocket URL - This is the URL of the websocket server you want to connect to. It starts wss://
  • WS: Terminate Connection - This is where you terminate an active websocket connection.
    • Name - The name of the connection that you entered when you started the connection.
  • WS: Send Message - This is how you send a message to the websocket server after you have started a connection.
    • Name - The name of the connection that you entered when you started the connection.
    • Message - This is the message you want to send to the server.

  • Listening to events - To listen to any response from the server, set an extension trigger on your button and use the name you entered when starting the connection.

    You can then use Trigger Pull Data and pull All to get the entire payload data.
  • All of the connections you start are viewable in the bridge.
